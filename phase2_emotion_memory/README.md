# Phase 2 - Emotion Memory

## 目標

Phase 1 只能針對單一畫面（單幀）辨識情緒，準確率約 60%，單幀預測容易受雜訊影響而跳動。

Phase 2 的目標是把「單點的情緒辨識」延伸成「一段時間內的情緒理解」，具體要回答三個問題：

1. 目前的情緒狀態是什麼（去除單幀雜訊後）？
2. 這個情緒狀態持續多久了？
3. 情緒是在變好、變差，還是穩定？

---

## 架構

```
攝影機畫面
    |
    v
Phase 1 模型推論（逐幀）
    |
    v
寫入 SQLite（保留完整機率分佈，不只存標籤）
    |
    v
EMA 平滑（去除單幀雜訊）
    |
    v
情緒摘要（目前狀態 / 持續時間 / 趨勢）
    |
    v
交給 Phase 3 使用
```

## 檔案說明

| 檔案 | 用途 |
|---|---|
| `config.py` | 資料庫路徑、平滑參數（EMA 係數、滑動視窗大小）、情緒效價對照表 |
| `database/db.py` | SQLite 存取層，負責寫入 / 查詢逐幀情緒紀錄 |
| `emotion_memory.py` | 核心演算法：EMA 平滑、情緒持續時間計算、趨勢判斷 |
| `capture_session.py` | 執行入口，讀取攝影機並呼叫 Phase 1 模型，逐幀寫入資料庫 |

## 使用方式

需要 `phase1_expression_detection/checkpoints/best_model.pth` 已存在（Phase 1 訓練完成的模型）。

```bash
cd phase2_emotion_memory
python capture_session.py
```

* 執行時會對著鏡頭偵測人臉並即時顯示情緒（畫面顯示的是平滑後的結果）
* 按 `ESC` 結束
* 結束後會印出這次的 `session_id`，資料存在 `database/emotion_history.db`

## 核心技術

* **EMA（指數移動平均）**：對逐幀機率分佈做平滑，濾掉單幀誤判造成的跳動，公式為
  `ema = α × 新機率 + (1-α) × 上一輪 ema`
* **機率加權趨勢判斷**：不是只看 argmax 標籤，而是把每個情緒類別轉成一個效價數值（`EMOTION_VALENCE`），用機率加權平均後比較「最近一段時間」與「再前一段時間」的差異，判斷情緒趨勢
* **SQLite**：儲存逐幀原始資料（含完整機率分佈），供之後分析或訓練更進階的時序模型使用

## 已知限制 / 之後可能的改進方向

* `EMOTION_VALENCE` 目前是手動設定的簡化假設（Happy=+1, Sad=-1...），未經實驗驗證，之後可能需要調整權重
* 趨勢判斷目前是純統計方法（滑動視窗比較），沒有用到機器學習模型；等累積足夠 session 資料後，可以考慮訓練 GRU/LSTM 取代，對應規劃中的 `memory_model/` 資料夾
* 目前假設單一使用者、單一人臉，沒有處理多人同時入鏡的情境
